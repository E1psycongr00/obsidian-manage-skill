# 🧠 DEEP DIVE 노트 규칙

## 역할

CORE 개념을 **디테일하게** 탐구하는 노트. **"왜 이렇게 동작하는가?"를 파고든 고민 과정**을 담는다. 단순 설명이 아니라, 탐구의 흔적이 보여야 한다.

## 작성 설계 가이드

- 목표: CORE 개념의 작동 이유와 전제, 한계를 탐구 흐름으로 해부한다.
- 권장 섹션(필수 아님): `핵심 질문 -> 설계 의도/배경 -> 분석/예시 -> 핵심 인사이트`
- 반드시 포함할 요소: `왜` 질문, CORE와의 연결, 단계적 분석 또는 반례
- 금지 요소: 백과사전식 요약만 반복, CORE와 무관한 독립 주제 전개

## 작성 보완 규칙

- "개요"로 시작하지 않고 핵심 질문 또는 문제 제기로 시작한다.
- 구조는 `핵심 질문 -> 메커니즘 분석 -> 인사이트 또는 열린 질문` 흐름을 권장한다.
- 주제에 따라 사례, 반례, 한계 분석, 트레이드오프를 선택적으로 배치한다.
- 작성자 관점과 판단을 드러낸다. 중립적 백과사전 문체만으로 끝내지 않는다.
- `core-concept`에 연결할 CORE 노트를 반드시 포함한다.
- 탐구 과정에서 함께 참조한 노트는 `related-note`에 연결한다. 이 필드는 CORE 외에도 REFERENCE, SOLUTION, EVIDENCE, REVIEW 등 어떤 노트든 받을 수 있다.
- 이미지/코드/타임라인 같은 시각 도구를 적극 사용해 깊이를 표현한다.
- 단계 번호와 헤더를 그대로 복제하지 않는다. 논리 흐름은 유지하되 제목은 주제에 맞게 조정한다.

## 작성 원칙

- ✅ 상세한 설명, 예시, 유도 과정
- ✅ CORE 개념과 반드시 연결
- ✅ 깊이 있는 탐구

## 적합한 내용

- CORE 개념의 상세 설명
- 구체적인 예시와 사례
- 수식 유도, 증명 과정
