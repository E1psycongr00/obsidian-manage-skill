# 📖 REFERENCE 노트 규칙

## 역할

**외부 자료 1개를 스크랩**하여 보존하는 노트.

## 작성 설계 가이드

- 목표: 원본 자료 1개를 손실 없이 보존하고 핵심 정보를 빠르게 재열람 가능하게 정리한다.
- 반드시 포함할 요소: 단일 명시적 출처, 원문 보존, 비해석적 요약
- 금지 요소: 분석/결론 추가, 주장 도출, 근거 없는 재해석, 여러 출처 묶음 요약

## 작성 보완 규칙

- 목적은 원본 보존이다. 분석/결론은 추가하지 않는다.
- REFERENCE 노트 1개는 source 1개만 다룬다.
- 스크립트/인용문은 가능하면 원문과 번역을 함께 둔다.
- 기사, 논문, 공식 문서, 블로그, 보고서 페이지처럼 온라인 원문이 핵심인 출처는 `source` 필드에 검증된 URL 1개를 반드시 기재한다.
- 책, 오프라인 문헌, 내부 문서처럼 안정적인 웹 원문이 없는 출처만 예외적으로 URL 대신 문헌 식별자 1개를 기재한다.
- URL은 문자열만 복사하지 말고 실제 브라우저 도구 또는 웹 도구로 열어 존재 여부, 문서 제목, 호스트가 맞는지 확인한 뒤 사용한다.
- 404, soft 404, 잘못된 리다이렉트, 존재하지 않는 페이지가 확인된 URL은 `source`에 적지 않는다.
- 원래 받은 URL이 죽어 있으면 검증 가능한 정식 URL, DOI URL, 아카이브 URL을 우선 찾고, 끝내 검증 가능한 URL이 없으면 없는 URL을 추정 작성하지 않는다.
- 파일 위치는 해당 모듈(또는 서브 모듈)의 `references/` 폴더를 사용한다.
- PDF/논문 원문/웹 스크랩 원문 등 참조 파일도 `references/` 폴더를 사용한다.
- 이미지 파일은 `references/`가 아니라 `attachments/` 폴더를 사용한다.
- 모듈 루트나 임의 폴더에 있는 REFERENCE 노트는 `references/`로 이동하고 링크를 점검한다.
- 여러 기사, 논문, 영상, 페이지를 한 노트에서 비교·큐레이션·공통점 요약하지 않는다.
- source가 2개 이상이면 source별로 REFERENCE를 분리하거나, 묶어서 주장/패턴을 만들 필요가 있으면 EVIDENCE로 전환한다.
- 헤더를 고정 템플릿처럼 복제하지 않는다. 자료 형식에 맞게 섹션 이름을 조정한다.

## 작성 원칙

- ✅ 원본 자료 보존
- ✅ 1노트 1출처
- ✅ 온라인 출처는 검증된 URL 명시
- ✅ 오프라인 출처는 문헌 식별자 명시
- ✅ URL 실접속 검증
- ✅ 핵심 요약 제공
- ❌ 여러 출처 묶음 정리 금지
- ❌ 존재하지 않는 페이지 URL 기입 금지
- ❌ 분석/결론 추가 금지 (그건 EVIDENCE)
