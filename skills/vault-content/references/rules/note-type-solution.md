# 🔬 SOLUTION 노트 규칙

## 역할

**문제 해결 과정**을 기록하는 노트.

## 작성 설계 가이드

- 목표: 재사용 가능한 해결 패턴을 문제 맥락부터 결과까지 남긴다.
- 전개 특징: 읽고 나면 무엇을 했는지보다 왜 이 해결이 먹혔는지가 남아야 한다.
- 반드시 살아 있어야 할 요소: 재현 가능한 절차, 선택 근거, 결과/효과
- 금지 요소: 문제 정의 없는 해답 제시, 결과 없는 구현 설명

## 작성 보완 규칙

- 시작은 문제 상황, 막히던 지점, 핵심 통찰 중 무엇이든 좋다. 다만 독자가 해결의 출발점을 빠르게 잡을 수 있어야 한다.
- 실패 케이스, 잘못된 접근, 선택의 분기점을 적절히 드러내 의사결정 근거가 보이게 한다.
- 해결 과정에는 실행 가능한 코드/절차와 핵심 동작 원리를 함께 적되, 단순 작업 일지처럼 나열하지 않는다.
- 결과는 수치, 로그, 재현 결과처럼 검증 가능한 형태로 제시한다.
- 이 해결이 어디까지 통하고 어디서부터 한계가 생기는지 짧게 남긴다.
- 필요하면 마지막에 짧은 판단/회고를 붙여 재사용 포인트를 남긴다.
- 섹션 제목을 그대로 복제하지 않는다. 문제 도메인에 맞춰 제목을 조정한다.

## 작성 원칙

- ✅ 해결 판단의 흐름이 보일 것
- ✅ 해결 과정의 논리적 흐름
- ✅ 재사용 가능한 솔루션 패턴
