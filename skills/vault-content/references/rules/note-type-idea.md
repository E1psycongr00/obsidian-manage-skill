# 💭 IDEA 노트 규칙

## 역할

아직 타입이 확정되지 않았거나 빠르게 보관해야 하는 아이디어를 정리하는 노트.
정보성 글, 임시 메모, 가설 스케치, 분류 보류 콘텐츠를 수용한다.

- 목표: 아이디어를 유실 없이 빠르게 기록하고, 나중에 승격 판단이 가능하도록 맥락을 남긴다.

## 작성 보완 규칙

- IDEA 노트는 임시/보류형 본문 구조와 승격 기준을 정의하는 문서다.
- 파일 배치(`idea/` 디렉토리)와 frontmatter 적용 여부는 구조 계약 문서를 따른다.
- 이 문서에서는 YAML 스키마를 다시 정의하지 않는다.
- 분류가 명확해지면 아래 기준으로 승격한다.
  - 개념 정제 완료: CORE/DEEP DIVE
  - 해결 절차 중심: SOLUTION
  - 근거/출처 기반 검증: EVIDENCE/REFERENCE
  - 개인 판단/회고 중심: REVIEW
- 헤더 구조는 자유롭게 구성한다.

## 작성 원칙

- ✅ 빠르게 기록하고 나중에 확장 가능해야 함
- ✅ 단편 메모, 탐색 질문, 키워드 덤프 허용
- ❌ 완성 문서처럼 과도하게 포장하지 않음

## 적합한 내용

- 아직 유형이 확정되지 않은 주제
- 프로젝트 초기 기획 메모
- 임시 아이디어와 탐색 질문 묶음
