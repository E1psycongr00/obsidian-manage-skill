# 📝 EVIDENCE 노트 규칙

## 역할

**주장을 뒷받침하는 증거**를 정리하고 결론을 도출하는 노트. 자료·실험·사실 등 **확인 가능한 근거**에서 추론한 결론을 담는다. 근거 없는 주관적 판단은 REVIEW다.

## 작성 설계 가이드

- 목표: 검증 가능한 근거를 바탕으로 특정 주장을 설득력 있게 제시한다.
- 반드시 포함할 요소: 출처가 있는 사실, 근거-주장 연결 논리, 문서 앞부분 결론 배치
- 금지 요소: 근거 없는 의견, 단순 정의 나열, 출처 미기재

## 작성 보완 규칙

- 맨 앞에 결론/주장을 두는 역피라미드 구조를 유지한다.
- "사실 나열"이 아니라 "주장 + 근거" 구조로 작성한다.
- 참조한 원본이 있다면 `ref-by`에 REFERENCE 노트를 연결한다.
- 여러 REFERENCE/source를 묶어 비교, 공통점 요약, 패턴 추출, 선택 기준 정리를 하면 EVIDENCE로 처리한다.
- 단순 개념 정의 중심으로 쓰지 않는다. 개념 정리는 CORE로 분리한다.
- 결론은 추측이 아니라 확인 가능한 근거에서 도출한다.
- 헤더 문구를 고정 템플릿처럼 복제하지 않는다. 주장과 근거에 맞게 제목을 조정한다.

## 작성 원칙

- ✅ 단일 또는 여러 자료에서 정보를 취합 (꼭 REFERENCE 노트가 아니어도 가능)
- ✅ 단순 개념 정의(이는 CORE의 영역)가 아닌, **특정 주장이나 논거를 바탕으로 도출한 통찰적 결론** 작성
- ✅ 확인할 수 있는 정보 기반 (신뢰할 수 있는 출처)
- ✅ 맨 앞에 취합 자료의 핵심 요약(결론)을 반드시 배치

## Reference vs Evidence

- **Reference**: 원본 자료 보존 및 핵심 요약 (데이터)
- **Evidence**: 하나 이상의 자료를 분석하고 결론 도출 (정보)
- 여러 source를 묶는 순간 REFERENCE보다 EVIDENCE 쪽이 우선이다.
