# 💡 CORE 노트 규칙

## 역할

핵심 개념을 **추상화**하여 **간결하게** 전달하는 노트.

## 작성 설계 가이드

- 보편적 개념의 본질을 짧고 명확하게 압축한다.
- 첫 섹션 규율: 본문 첫 섹션은 반드시 본문을 요약한 핵심 개념들이나, 아니면 정의와 같은 것으로 시작한다.
- 반드시 포함할 요소: 개념의 경계(무엇이고 무엇이 아닌지), 핵심 키워드, 재사용 가능한 요약 문장, 오해 방지 포인트
- 금지 요소: 장문의 사례 설명, 코드/실험 상세 전개, 복수 주제의 혼합

## 작성 보완 규칙

- 예시와 코드는 최소화하고, 상세 전개는 DEEP DIVE로 넘긴다.
- 초보자가 자주 혼동하는 지점을 1개 이상 분리해 정리한다.
- 오해 방지 섹션 제목은 `자주 헷갈리는 포인트`, `오해하기 쉬운 부분`처럼 주제에 맞게 조정한다.
- 확장 탐구 노트가 있다면 `related-document`를 연결한다.
- 헤더 문구를 그대로 복제하지 않는다. 같은 의도를 유지한 채 제목은 주제에 맞게 재구성한다.

## 작성 원칙

- ✅ 한눈에 파악 가능하게 간략하게
- ✅ 개념의 핵심만 다룸
- ❌ 디테일한 설명, 긴 예시 금지
- ❌ 여러 개념을 나열하는 것은 가능하지만 여러 개념의 주제를 동시에 다루는 건 금지

## 적합한 내용

- 보편타당한 사실 또는 **불변하는 개념 자체의 정의**
- 수학/과학/공학 이론의 핵심 원리
- 복잡한 정보를 **하나의 추상화된 원형**으로 압축한 결과물

> ⚠️ 인터뷰, 뉴스, 개인 체험담에서 도출한 결론은 CORE가 아닌 `📝 EVIDENCE`로 작성한다.
