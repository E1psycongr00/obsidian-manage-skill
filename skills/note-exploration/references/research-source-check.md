# Research Source Check

탐색 단계의 조사는 `방향을 선명하게 하기 위한 최소 조사`여야 한다.
자료가 많아지는 것보다, 실제 존재하는 믿을 만한 자료 몇 개가 더 중요하다.

## 출처 우선순위

가능하면 위에서 아래 순서로 우선한다.

1. 공식 문서, 기관 보고서, 논문 원문, 데이터 원출처
2. 저자와 맥락이 분명한 전문 분석, 인터뷰, 실무 회고
3. 실제 사례를 담은 기사, 회사 블로그, 검증 가능한 발표 자료

아래는 기본적으로 경계한다.

- 제목 낚시형 요약 사이트
- 출처가 흐린 재가공 블로그
- SEO용 목록 글
- 검색 결과 스니펫만 보고 추정한 URL

## URL 검증 체크

웹사이트를 예시나 출처 후보로 직접 언급하려면 최소한 아래를 확인한다.

- 실제로 페이지가 열리는가
- 호스트가 기대한 사이트가 맞는가
- 문서 제목이 검색 의도와 맞는가
- 시간 민감 주제라면 날짜가 너무 오래되지 않았는가

하나라도 애매하면 그 링크는 확정 출처처럼 쓰지 않는다.

## 탐색 단계에서의 사용법

- 방향 하나당 검증된 자료 0~2개면 충분한 경우가 많다.
- 조사한 사실은 `이 방향이 왜 살아 있는가`를 설명하는 데만 쓴다.
- 자료 요약이 길어지면 방향 탐색이 아니라 리서치 노트가 되므로 줄인다.
- 아직 검증 전인 사례는 `후보`라고 표시하고 단정 문장으로 쓰지 않는다.

## 짧은 메모 형식

필요할 때만 아래처럼 아주 짧게 메모한다.

- 검증됨: [사이트/문서 이름] - 이 방향에서 무엇을 보여 주는가
- 후보: [사례 이름] - 아직 존재 여부 또는 적합성 확인 필요
