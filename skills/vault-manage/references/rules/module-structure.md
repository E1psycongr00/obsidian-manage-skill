# 📦 모듈 구조 규칙

AI Agent가 모듈을 생성하거나 관리할 때 따라야 하는 규칙입니다.

---

## 1. 모듈 개념

**모듈**은 독립적으로 존재하면서도 조합 가능한 **지식 단위**입니다.

**핵심 가치**: 레고 블록처럼 언제든 분리하거나 다른 곳에 붙일 수 있는 **확장성**

---

## 2. 모듈 유형

메인 모듈은 PARA 위치에 따라 3가지 유형으로 나뉩니다.

| 유형 | 아이콘 | 위치 | 특성 |
|------|--------|------|------|
| **Projects** | 🚀 | `01. Projects/` | 명확한 목표와 **기한**이 있음 |
| **Area** | 🌱 | `02. Area/` | 기한 없이 **지속적으로 관리**해야 하는 책임 |
| **Resources** | 📚 | `03. Resources/` | 개인적 **흥미나 지식 정리** 용도 |

> ⚠️ **주의**: 📚 Resources 메인 모듈은 PARA의 `03. Resources/` 폴더에 위치하는 메인 모듈입니다. 모듈 내부의 `references/` 폴더(REFERENCE 노트/원문 자료 저장용)와 혼동하지 마세요.

### 🚀 Projects (프로젝트 모듈)

**위치**: `01. Projects/`
**특성**: 명확한 목표와 **기한**이 있는 프로젝트

**폴더 구조**:
```
🚀 프로젝트명/
├── 🚀 프로젝트명.md       # 인덱스 노트 (필수)
├── attachments/           # 모듈 전용 이미지
├── references/            # REFERENCE 노트/원문 자료
├── idea/                  # IDEA/raw 임시 노트 (선택)
├── 💡 코어노트.md
└── 📦 서브모듈/           # 서브 모듈 (선택)
```

**인덱스 노트 필수 내용**:
- 왜 이 프로젝트를 시작했나 (동기)
- 목표 및 완료 조건
- 마감일 / 진행 상태
- Map of Content

---

### 🌱 Area (영역 모듈)

**위치**: `02. Area/`
**특성**: 기한 없이 **지속적으로 관리**해야 하는 책임 영역

**폴더 구조**:
```
🌱 영역명/
├── 🌱 영역명.md           # 인덱스 노트 (필수)
├── attachments/           # 모듈 전용 이미지
├── references/            # REFERENCE 노트/원문 자료
├── idea/                  # IDEA/raw 임시 노트 (선택)
├── 💡 코어노트.md
└── 📦 서브모듈/           # 서브 모듈 (선택)
```

**인덱스 노트 필수 내용**:
- 왜 이 영역을 관리하나 (책임)
- 유지 기준 / 품질 목표
- 관련 루틴 (선택)
- Map of Content

---

### 📚 Resources (자료 모듈) - 메인 모듈

**위치**: `03. Resources/`
**특성**: 개인적 **흥미나 지식 정리** 용도

> 📌 이것은 **PARA의 Resources 폴더에 위치하는 메인 모듈**입니다. 모듈 내부의 `references/` 폴더(REFERENCE 노트/원문 자료 저장)와 다릅니다.

**폴더 구조**:
```
📚 주제명/
├── 📚 주제명.md           # 인덱스 노트 (필수)
├── attachments/           # 모듈 전용 이미지
├── references/            # REFERENCE 노트/원문 자료
├── idea/                  # IDEA/raw 임시 노트 (선택)
├── 💡 코어노트.md
└── 📦 서브모듈/           # 서브 모듈 (선택)
```

**인덱스 노트 필수 내용**:
- 왜 이 주제에 관심을 갖게 됐나 (계기)
- 학습 / 정리 목적
- 핵심 개념 (선택)
- Map of Content

---

### 📦 서브 모듈 (Sub Module)

**위치**: 메인 모듈 또는 다른 서브 모듈 내부
**용도**: 상위 모듈의 세부 주제를 체계적으로 관리

**폴더 구조**:
```
📦 서브모듈명/
├── 📦 서브모듈명.md       # 인덱스 노트 (필수)
├── attachments/           # 서브 모듈 전용 이미지
├── references/            # REFERENCE 노트/원문 자료 (필수)
├── idea/                  # IDEA/raw 임시 노트 (선택)
└── 관련노트들.md
```

**인덱스 노트 필수 내용**:
- 왜 이 서브 모듈이 필요한가?
- 어떤 노트들을 관리하는가?
- 상위 모듈과의 관계

**REFERENCE 노트 위치 규칙**:
- `note-type: REFERENCE` 노트는 반드시 해당 모듈(또는 서브 모듈)의 `references/` 폴더에 둔다.
- PDF/논문 원문/웹 스크랩 원문 등 참조 파일도 `references/` 폴더에 둔다.
- 모듈 루트/`idea/`/기타 폴더에 REFERENCE 노트를 두지 않는다.

**IDEA 노트 위치 규칙**:
- 임시/보류 상태의 IDEA 노트는 해당 모듈(또는 서브 모듈)의 `idea/` 폴더에 둔다.
- IDEA는 raw markdown을 기본값으로 허용한다.
- IDEA를 다른 타입으로 승격하면 모듈 루트 또는 타입 규칙에 맞는 위치로 이동한다.

## 5. 모듈 컨텍스트 원칙

모듈 컨텍스트의 정본은 모듈 루트의 인덱스 노트다. 별도 `.context.md` 파일은 만들지 않는다.

### 기본 원칙

- 별도 `.context.md`는 만들지 않는다.
- AI가 맥락을 파악할 때는 인덱스 노트 -> `Map of Content` -> 관련 하위 노트 순서로 읽는다.
- 노트 목록은 별도 표를 중복 작성하지 않고 `Map of Content`와 실제 폴더 구조를 기준으로 삼는다.

## 8. 이미지(attachments) 관리 규칙

### 저장 위치
- 모든 이미지는 **해당 모듈의 `attachments/` 폴더**에 저장
- 서브 모듈의 이미지는 서브 모듈 내 `attachments/`에 저장
- PDF 등 참조 파일은 `attachments/`가 아닌 `references/`에 저장
- 모듈 경계를 넘어 이미지를 공유하지 않음 (독립성 유지)

### 파일명 규칙

**형식**: `[노트명]_[설명].[확장자]`

| 구성 요소 | 설명 | 예시 |
|-----------|------|------|
| 노트명 | 이미지가 사용되는 노트 (이모지 포함) | `💡 FSM` |
| 설명 | 이미지 내용 간략 설명 | `상태전이도` |
| 확장자 | 이미지 형식 | `.png`, `.jpg`, `.svg` |

**좋은 예시**:
```
💡 FSM_상태전이도.png
🧠 HFSM 깊이탐구_계층구조예시.png
🔬 문제해결_플로우차트.svg
```

**허용 예외**:
- 여러 노트에서 공통 사용: `diagram_[설명].png`
- 스크린샷: `screenshot_[날짜]_[설명].png`

### AI 이미지 확인 가이드

1. **이미지 위치 추론**: 노트에서 `![[이미지명]]` 링크 발견 시, 해당 모듈의 `attachments/` 폴더에서 검색
2. **이미지 확인 방법**: `view_file` 도구로 이미지 파일 직접 확인 가능
3. **파일명으로 맥락 파악**: 파일명의 노트명과 설명으로 이미지 용도 추론

### 체크리스트

이미지 추가 시:
- [ ] 해당 모듈의 `attachments/` 폴더에 저장
- [ ] 파일명 규칙 준수 (`[노트명]_[설명].[확장자]`)
- [ ] 노트에서 `![[이미지명]]` 형식으로 링크
